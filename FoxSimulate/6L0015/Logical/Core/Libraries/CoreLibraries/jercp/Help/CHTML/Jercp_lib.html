jercp_lib

The jercp_lib provides some recipe handling functions.

 

Mainly it is intended to load & save recipe data, which is stored in a File in csv Format.

This has the advantage that the recipe data could be edited using Excel.

 

The Library can work with single Structures & Structure Arrays. The user can configure if the recipe data should be stored on a variable - e.g. structure / structure array - or if it should be stored in an allocated memory. In some applications it is essential to have all recipe data available in memory because it must be accessed very quickly. In that case, if the whole recipe data exceeds a size of 32KByte, a dynamic handling is necessary.

 

Optional possibilities:

    - Edit of the Recipe data (Either by variable names or clear text - from a seperate csv file - in different languages)

    - Copy from Recipe Index X a defined number Y of recipes to a Destination Index Z

    - Add a Recipe (only when working with dynamic allocated memory)

    - Create a Recipe Name List (If a recipe contains its name, a List to scroll in could be created)

    - Controlled referencing of a recipe pointer to available recipes (prevent wrong referencing)
